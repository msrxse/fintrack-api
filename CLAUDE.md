# Project: FinTrack API — Teaching Project

## Teaching Approach
1. Discuss the concept in chat first
2. User asks questions until they understand
3. Only then write the file
- No code comments in files — explanations happen in chat only
- One file at a time — never batch multiple files without discussion
- User is learning, not watching — keep them in the driver's seat

## Stack
- Python 3.11, FastAPI, PostgreSQL (Docker), SQLAlchemy, Alembic, PyJWT, Pydantic v2, Poetry

## Key Preferences
- Poetry for dependency management (not pip/requirements.txt)
- No faker library — seed data written by hand
- No inline code comments unless explicitly asked
